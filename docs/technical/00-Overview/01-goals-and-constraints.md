# PTB-OVR-001 — اهداف، قیود و دامنه

## 1) اهداف (Goals)
پروژه باید:
1) **Local‑First** باشد: ابزارها بدون نیاز به اینترنت جهانی قابل استفاده باشند.
2) روی **VPS ایرانی** پایدار باشد و در شرایط فیلترینگ/اختلال هم UX قابل قبول ارائه کند.
3) برای ابزارهای نیازمند داده/منبع خارجی، **Hybrid** طراحی شود: خروجی با آخرین داده معتبر و پیام شفاف.
4) از نظر UX، برای کاربران فارسی‌زبان (اعداد فارسی، راست‌چین، موبایل) **بدون اصطکاک** باشد.
5) از نظر SEO، ساختار دسته‌بندی ابزارها و صفحات راهنما، **قابل ایندکس و خوشه‌ای** باشد.
6) مسیر درآمدی:
   - ابزارهای عمومی: **رایگان**
   - ابزارهای تخصصی: **پولی** (Online‑Required) با کنترل دسترسی
   - CTA برای سفارش ابزار اختصاصی سازمانی

## 2) قیود (Constraints)
- اینترنت جهانی در ایران ممکن است قطع/کند شود.
- تحریم‌ها می‌توانند سرویس‌ها و CDNها را غیرقابل دسترس کنند.
- اصل حاکم: **هیچ وابستگی runtime خارجی** مگر با استثنای کنترل‌شده و ثبت‌شده.

## 3) خارج از دامنه (Non‑Goals)
- ساخت سیستم پرداخت/Subscription در همین فاز اصلاحی اجباری نیست؛ فقط طراحی معماری و آماده‌سازی مسیرها لازم است.
- اپ اندروید در فاز اصلاحی اجباری نیست؛ آماده‌سازی PWA و سیاست‌های cache کافی است.

## 4) معیارهای پذیرش کلان
- حداقل یک مسیر کاربری کامل (مثلاً Loan + DateTools) باید در حالت آفلاین ۱۰۰٪ قابل استفاده باشد.
- هر ابزار Hybrid باید:
  - تاریخ آخرین بروزرسانی داده را نمایش دهد
  - در نبود اینترنت/منبع خارجی با “آخرین داده” کار کند یا پیام توقف بدهد
