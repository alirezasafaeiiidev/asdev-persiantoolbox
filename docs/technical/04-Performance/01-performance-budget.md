# PTB-PERF-001 — بودجه Performance و اهداف اجرایی

## هدف
لود سریع و تعامل روان روی موبایل‌های متوسط و اینترنت ایران.

## 1) بودجه باندل
- باندل اولیه باید حداقل باشد.
- ابزارهای سنگین باید فقط در route خودشان lazy شوند.

## 2) فونت‌ها
### باید (MUST)
- فونت‌ها WOFF2 باشند.
- preload صحیح انجام شود.
- fallback font stack مناسب باشد تا CLS ایجاد نشود.

## 3) انیمیشن
- انیمیشن‌های سنگین در فرم‌های ورودی باید حداقلی شوند.
- برای hover و transition ساده از CSS استفاده شود.
- برای route های سنگین از dynamic import برای motion استفاده شود.

## 4) Service Worker
- مسیرهای Offline‑Guaranteed در Shell cache باشند.
- مسیرهای Pro/Online‑Required نباید cache شوند.

## معیار پذیرش
- TTI/INP در موبایل قابل قبول باشد (تعامل اولیه بدون لگ محسوس).
- صفحات ابزارهای PDF باعث افزایش شدید JS اولیه نشوند.
