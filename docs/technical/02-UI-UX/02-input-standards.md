# PTB-UX-002 — استاندارد ورودی‌ها (Numeric/Money/Date)

## هدف
ورودی‌های عددی و تاریخ باید:
- روی موبایل ساده و سریع باشند
- اعداد فارسی/عربی را بدون خطا قبول کنند
- خطاها را دقیق و فیلد‑محور نمایش دهند
- از خطاهای فرمت و تاریخ نامعتبر جلوگیری کنند

---

## 1) Numeric / Money Input

### باید (MUST)
- `inputMode="numeric"`
- `dir="ltr"` برای بخش عددی
- پشتیبانی از اعداد فارسی/عربی (normalize)
- پشتیبانی از جداکننده هزارگان (`,`, `٫`, `٬`, فاصله)
- Validation فیلد‑محور:
  - min/max
  - required
  - پیام خطای کوتاه و دقیق

### ترجیحاً (SHOULD)
- فرمت‌دهی حین تایپ (thousand separator) با حفظ caret
- نمایش “معادل حروفی” برای مبالغ بزرگ (برای کاهش خطای انسانی)
- کلیدهای میانبر: Enter برای محاسبه، Esc برای پاک کردن (در صورت منطقی بودن)

### خروجی فنی
یک کامپوننت مشترک بسازید:
- `NumericInput`
- `MoneyInput` (wrapper با واحد تومان/ریال)

**API پیشنهادی:**
```ts
type NumericInputProps = {
  label: string;
  value: string;
  onValueChange: (raw: string, parsed: number | null) => void;
  min?: number;
  max?: number;
  required?: boolean;
  hint?: string;
  error?: string;
  unit?: "IRR" | "TOMAN" | "PERCENT" | "MONTH";
};
```

## 2) Date Input (Jalali/Gregorian)

### باید (MUST)
- ورودی تاریخ باید ساختاریافته باشد (روز/ماه/سال) یا یک DatePicker قابل اعتماد.
- اعتبارسنجی باید “preventive” باشد: انتخاب روزهای نامعتبر مجاز نباشد.
- پیام خطا باید نزدیک همان فیلد نمایش داده شود.

### ترجیحاً (SHOULD)
- روزهای قابل انتخاب بر اساس ماه/سال داینامیک شود.
- سال range مشخص داشته باشد (مثلاً 1250 تا 1500 شمسی) یا راهنمایی شود.

---

## معیار پذیرش
- کاربر بتواند اعداد فارسی را کپی‑پیست کند و خروجی درست بگیرد.
- در موبایل، کیبورد عددی باز شود.
- برای تاریخ، کاربر نتواند تاریخ نامعتبر بسازد (یا حداقل سریع اصلاح شود).
