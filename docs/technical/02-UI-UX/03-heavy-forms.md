# PTB-UX-003 — فرم‌های سنگین (Loan/Salary) و کاهش اصطکاک

## هدف
فرم‌های سنگین باید:
- قابل فهم باشند
- خطاها سریع مشخص شوند
- خروجی قابل اعتماد و قابل اشتراک/دانلود باشد

## استاندارد طراحی فرم (باید)
1) هر فرم باید یک CTA اصلی داشته باشد: “محاسبه”
2) ورودی‌ها باید گروه‌بندی شوند:
   - ورودی‌های اصلی
   - تنظیمات پیشرفته (Advanced)
3) خطاها باید فیلد‑محور باشند (نه فقط پیام کلی)
4) خروجی باید breakdown داشته باشد (جزئیات قابل مشاهده)

## الگوی پیشنهادی برای Salary
به‌جای یک فرم بزرگ، یک Wizard مرحله‌ای بسازید:
1) انتخاب Mode + ورودی‌های اصلی
2) ورودی‌های تکمیلی (مزایا/کسورات/وضعیت)
3) خروجی + دانلود + سناریو

## معیار پذیرش
- کاربر در موبایل بدون اسکرول بی‌پایان بتواند ورودی‌های اصلی را تکمیل کند.
- Advanced به‌صورت پیش‌فرض بسته باشد.
- هر خطا دقیقاً به یک فیلد متصل باشد (aria-describedby).
