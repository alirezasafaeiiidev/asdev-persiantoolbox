# استانداردهای پروژه Persian Tools

> آخرین به‌روزرسانی: 2026-02-12

## 1) اصول بنیادین

- **Local-first**: پردازش‌ها تا حد امکان در مرورگر انجام شود.
- **بدون وابستگی خارجی در runtime**: عدم استفاده از CDN برای هسته.
- **RTL و فارسی‌محور**: سازگاری کامل با راست‌به‌چپ.
- **حریم خصوصی پیش‌فرض**: داده‌ها حداقلی و شفاف.

## 2) استانداردهای فنی

- `strict` در TypeScript فعال است.
- ماژول‌ها کوچک و قابل تست باشند.
- وابستگی‌های dev حداقلی و قابل توجیه باشند.

## 3) طراحی و UI/UX

- استفاده از توکن‌های طراحی در `shared/constants/tokens.ts`.
- کلاس‌های منطقی RTL (`start/end`) به‌جای `left/right`.
- انیمیشن‌ها سبک و مطابق `prefers-reduced-motion`.

## 4) دسترس‌پذیری (WCAG 2.1 AA)

- `aria` مناسب برای فرم‌ها و وضعیت‌ها.
- ناوبری کیبورد برای مسیرهای اصلی.
- کنتراست حداقل AA.

## 5) SEO و محتوا

- متادیتا و OG برای تمام صفحات ابزار.
- داده ساخت‌یافته برای صفحات کلیدی.

## 6) PWA و آفلاین

- Service Worker در `/sw.js`.
- صفحه آفلاین `/offline`.
- bump نسخه `CACHE_VERSION` برای هر انتشار.

## 7) تست و کیفیت

- حداقل پوشش تست برای ابزارهای اصلی ≥ 85%.
- گیت قراردادی سبک:
  - `pnpm ci:contracts`

## 8) مستندسازی

- تغییرات بزرگ باید در `docs/` منعکس شود.
- README باید با وضعیت واقعی هم‌راستا باشد.
