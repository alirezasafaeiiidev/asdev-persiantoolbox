# Contributing

## Branching

Use `type/short-description` naming, for example `chore/update-docs`.

## Commits

Use Conventional Commits (example: `docs: update setup notes`).

## Pull Requests

- Keep PRs small and focused.
- Include a clear summary, rationale, and validation evidence.
- Link related ADR/standard when change is policy- or standards-related.

## DCO Requirement

- All contributions must include a valid `Signed-off-by` trailer in every commit message.
- Policy reference: `DCO.md`.

## CLA Hybrid

- External contributions are governed by DCO + CLA hybrid policy.
- Individual CLA: `docs/licensing/cla-individual.md`
- Corporate CLA: `docs/licensing/cla-corporate.md`
